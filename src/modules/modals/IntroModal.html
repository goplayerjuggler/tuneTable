“Tune table” is a tool for browsing, organising, and exploring collections of traditional and folk tunes. You can view sheet music, filter tunes, browse notes and references, and sort lists in (as far as I know) a novel way. The sorting involves an automatically-generated contour, which is a type of diagram based on the beginning of a tune – see “Sorting and contours” below for more info.

This is an alpha-stage personal project, not a polished product. Several features are incomplete or untested; in general there will be bugs. Also some features, particularly editing and importing tunes, require some technical know-how. But feel free to play around with it; you won’t break anything!

Tune lists – server and local

When you first open Tune table you may be asked to choose a tune list. There are two kinds:

Server lists are read-only snapshots compiled and hosted online. Any edits you make are saved automatically as a local copy in your browser.

Local lists are stored in your browser’s local storage. They will be lost if you clear your browser data, switch browsers or devices, or if the browser happens to free up space for some reason or other. So if you use local lists, do make backups by exporting your tune list via the Edit menu (“Copy tune data”).

Local lists can be used to import tunes in ABC format , or to import users’ tunebooks and sets from thesession.org. For example, you could try creating an empty local list and then importing all the tunes from one of Henrick Norbeck’s traditional music collections.

You can switch between lists at any time using the Tune lists button in the page footer or the Edit menu.

Sorting and contours

Clicking the first column header cycles through three top-level sorting modes: by either the tune’s rhythmic type (i.e. jig, reel, mazurka, hornpipe …), its time signature, or its name. After the top-level sort, tunes are sorted by their contour.

A tune’s contour (shown as a small graphic next to the title) is its melodic outline: the shape of the melody relative to the tonic (home note), independent of key or mode (major, minor, mixolydian…). The contour graphic is generated automatically from the tune’s incipit (the first few notes of the tune).

Contour sort groups tunes together when their contours start similarly, whatever key they happen to be in. For example, the following five jigs in my default tune list are grouped together (because they start similarly): Leave it down easy, Wellington’s Advance, Connie the Soldier, Old Man Dillon, and Darby The Driver. Comparing these tunes’ contours with their neighbours may help you understand the sorting sytem.

I find having tunes sorted this way makes it easier to find a tune whose name I’ve forgotten. I simply look it up in the list, based on how it starts. It’s just like how one used to look up words in printed dictionaries.

Score viewer

Clicking on a tune’s name (shown as a link when a score is available) opens a score viewer. From there you can:

  • read the full score, with page navigation for multi-page scores;
  • transpose up or down by semitone steps;
  • switch bar length for certain tune types – see About my collection of tunes;
  • view the raw ABC notation source;
  • navigate between multiple settings of the same tune, if there are any in the current tune list;
  • view a whole set of tunes as a single continuous score.

Changes made in the score viewer can be saved back to your local tune list.

Set lists

I use the set lists feature to group tunes into ordered sets – for a session, a class, a gig, or any other occasion. You can create and manage sets from the Edit menu.

Sets can be printed to PDF (just the incipits; not the full scores), or viewed as a continuous full score in the score viewer. Set list data is saved alongside your tune list; the same backup advice made above (see “Tune lists – server and local”) applies.

About my collection of tunes
  • The default tune list is my personal selection drawn from Irish, Scottish, French, Scandinavian, Québécois, and other traditional music. Although most of the scores are other people’s work that I’ve taken from thesession.org, a fair few of the scores (also known as settings) were written by me.
  • For some tunes I’ve worked on, I’ve added external references to videos or audio recordings. A few examples: The First Draft (a mazurka by Siobhán Peoples); two different “Tommy Potts-esque” settings of the jig Rambling Pitchfork (firstly by Jimmy Power, and secondly by Paddy O’Brien & Nathan Gourley); the bourrée à Chandezon, a “bourrée à 3 temps” with several references, …. I’ve also added some cross-references between tunes, often when an audio reference is for a set of tunes.
  • There may be some errors in the scores. In any case, I don't want to build an authoritative reference; I’m by no means an expert. I'm just aiming to make a personal reference library here.
  • Most of the text is in English, but notes and titles for some French tunes appear in French.
  • Bar lengths for some tunes are non-standard. I prefer reels to be written in 4/4 with semiquavers (sixteenth notes) rather than the more common notation with quavers (eighth notes). The bar-length switcher in the score viewer lets you toggle between the two representations. Similar conventions apply to jigs and hornpipes.
